Specification and Data

Aim

Given

  • labeled dataset consisting 16,000 labeled observations, 166 features and a binary target (right-most column), and
  • scoring dataset consisting of 9,000 unlabelled observations (the 166 features only).

you are asked to develop a classification model (or models) that predict the labels for the scoring dataset. Your model performance will be evaluated using two metrics: accuracy using accuracy_score and area under the roc curve using roc_auc_score.

This is a 100% data-focused task so you get no context and

  • most of the categorical variables have already been encoded using OrdinalEncoder.
  • column labels have been removed.
    • in the labeled dataset, the rightmost column is the binary target.
    • The scoring dataset, only contains the 166 features.

Deliverables

You need to upload three files, these are uploaded separately

  • Model.ipynb
  • pred_accuracy.csv
  • pred_roc_auc.csv

Deliverable Model.ipynb

Model.ipynb is a juypter-lab notebook (IPYNB) with

  • Ensure that this is executable without editing in a folder containing a subfolder data which in turn contains the two datafiles labeled.csv.gz and scoring.csv.gz. (I will create the subfolder data and its contents.)
  • Executing all cells in notebook should generate the both pred_accuracy.csv.gz and pred_roc_auc.csv.gz.
  • For cells that contain code that takes a log time to run, but whose output you have already captured and used (think hyperparameter tuning) you can (should) define a boolean flag, say RUN_ALL, at top of notebook and wrap cell code in an if block like:
1
2
if RUN_ALL:
    # long running code goes here
  • To purpose of the notebook is to document your steps to develop an optimal model. As in the hyperparameter tuning task, you can use any of the sklearn classifiers (except SVM and neural networks). In particular I am interested in seeing some effort in the following:
    • cross validation when making model decisions
    • feature engineering (encoding, transformation, selection)
    • model selection
    • ensemble models — bagging and boosting
Upload notebook Model.ipynb

Deliverable pred_accuracy.csv.gz

This CSV.GZ file (compressed used using option compression='gzip' in to_csv call.) should have (at least) a column called target containing the 9,000 predicted labels for observations in the same order as given in the scoring dataset.

Before the assignment due date any submitted files will be grading using a subset (500) of the predictions. This is, a random subset of your 9,000 predictions will be compared with the true labels and scored using the accuracy score.

After the assignment due date the entire file will be used when computing the accuracy score and the resulting grade.

Upload predictions pred_accuracy.csv.gz

Deliverable pred_roc_auc.csv.gz

This file is similar to pred_accuracy.csv.gz, except that the area under the roc curve score is used to evaluate the model.

Since this metric needs the probability of being in a class (via predict_proba) rather than the actual prediction (via predict), so CSV file should have (at least) a column called pr_target containing the probability of observation being in class 0.

Note: You don't have to use to same model when generating the predictions to maximise accuracy and when maximising roc_auc_score.

Upload predictions pred_roc_auc.csv.gz
