Feature Engineering - Churn

In this practical we will try to keep everything else fixed and focus on one single aspect of the data mining pipeline --- feature engineering, or more specifically feature extraction and feature selection. (Yes, I'm aware of the inconsistently between "single" and the use of "and". :-/)

You starting point is the notebook which:

  • Loads the Churn dataset and performs basic cleaning and preparation (as we did in the previous practical).
  • Creates a single feature. This feature is intentionally silly, it sole purpose is to illustrate the process.
  • Specifies the desired metric - in this case, recall metric.
  • Carries out training of a few classifiers.
  • Applies automatic feature selection and replays model training using set of classifiers to obtain final metric.

You job this week is to create new features to see if it is possible to find features to better explain the target and hopefully result in a better score (as defined in the notebook).

To keep this exercise as realistic as possible I'm not going to tell you which features will be of benefit, nor how much (if any) the chosen metric can be improved.

Suggestions:

  • Adding features related to State appear to be a good idea.
  • I'm not saying that such features will improve the score. This is for you to determine.

  • However, don't spend all your time adding State related features - it may have no effect in improving the recall metric.

  • Best strategy is to develop multiple features using multiple lines of attack --- features that might seem plausible might not be of benefit when applied to the data.

So don't get tunnel-vision and unnecessarily limit your options.

  • Don't spend too much time on this - the practical is not worth much. I considered setting the upload time to be end of practical but decided against this as I don't like benefiting "fast over correct" programming.
